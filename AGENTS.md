# AGENTS.md

This file provides guidance to coding agents like Claude Code (claude.ai/code) when working with code in this repository.

## Project

Static website for the Engineering Kiosk podcast (engineeringkiosk.dev).
Astro 6 site with Tailwind v4, deployed to Netlify.
A separate Go CLI under `website-admin/` syncs external data (RSS feed, German tech podcasts, software-engineering games/movies, tag metadata, Netlify redirects) into the Astro content collections.

Node `>=24` (see `.nvmrc`).
German is the primary content language.

## Commands

Frontend (run from repo root):

- `make init` — `npm install`
- `make run` — `astro dev` (dev server)
- `make build` — `astro build` → `dist/`
- `make test-javascript` / `npm test` — vitest (run mode). Single test: `npx vitest run path/to/file.test.js` or `npx vitest run -t "name"`.
- `make prettier` — format everything

Go CLI (`website-admin/`, run from that directory):

- `make build` → `./website-admin`
- `make test` — `go test -v -race ./...`
- `make staticcheck`, `make vet`, `make code-coverage`
- `./website-admin sync <subcommand>` and `./website-admin podcast <subcommand>` — see `cmd/cmd_sync.go`, `cmd/cmd_podcast.go`, `cmd/cmd_tags.go`. Most subcommands are wired into GitHub Actions in `.github/workflows/sync-*.yml` and `update-podcast-episodes-from-rss.yml`.
- Top-level shortcut: `make update-german-tech-podcasts` calls the prebuilt `./website-admin/website-admin sync german-tech-podcasts`.

## Architecture

### Astro content collections

Schemas are centralised in `src/content.config.ts`.
Each collection loads from `src/content/<name>/`:

- `podcast` — episode markdown (frontmatter incl. chapters, speakers, transcript pointer, platform URLs). Body is the German show notes.
- `blog` — `.md`/`.mdx` posts with `thumbnail` + `headerimage` (image sizes documented in `README.md`).
- `meetup-alps`, `meetup-rhine-ruhr` — share `meetupSchema`. New meetup regions = new collection + new pages under `src/pages/meetup/<region>/`.
- `germantechpodcasts`, `awesome-software-engineering-games`, `awesome-software-engineering-movies` — JSON, regenerated by `website-admin sync …`. Treat the JSON as derived: edit upstream sources or the sync command, not the files.

Static singletons (tag descriptions, team members, badge links, "best of" episode list, podcast metadata) live in `src/data/*.json`.
Per-episode transcripts live in `src/data/transcripts/`.

### Page routing

`src/pages/` mirrors the public URL space.
Episode pages are generated under `src/pages/podcast/episode/`, blog posts under `src/pages/blog/post/`, meetups per region under `src/pages/meetup/<region>/`.
`trailingSlash: 'always'` and `output: 'static'` are set in `astro.config.mjs`.

### Sitemap

`@astrojs/sitemap` is wired in `astro.config.mjs` with helpers from `scripts/sitemap-lastmod.mjs`:

- `excludeFromSitemap` lists path substrings to drop (short-lived meetup promo pages, linktree).
- `buildLastmodMap()` is computed once per build (sync `serialize` hook constraint).
- Each URL gets one `<image:image>` entry from the rendered `og:image` of the built page — keep this contract in mind when adding new page types or changing OG image emission.

### Podcast player

Custom fork of Podigee's player at `EngineeringKiosk/podigee-podcast-player` plus a vendored minified `player.js` (embedly) under `public/js/`.
Upgrade by replacing the file — see README "Podcast player" section.

### Netlify redirects

`netlify.toml` is enormous and machine-managed for `/episodes/<n>` → canonical episode URLs.
Generated by `website-admin podcast netlify-redirect` (see `cmd/cmd_podcast_netlify_redirect.go`).
Don't hand-edit episode redirects; regenerate.

### Go CLI layout

`website-admin/cmd/` uses cobra.
Each `cmd_*.go` file is one subcommand with a paired `_test.go`.
Helpers in `website-admin/episode/` and `website-admin/utils/`.
The CLI reads/writes the Astro content collections directly (`src/content/...`), so its tests double as the schema contract for sync output.

## Conventions worth knowing

- Prettier with `prettier-plugin-astro` is the formatter (`.prettierrc`); CI enforces it (`.github/workflows/prettier.yml`).
- Tailwind v4 via `@tailwindcss/vite`; no `tailwind.config.js` — config is CSS-first in `src/styles/`.
- Tests: vitest for JS (`src/scripts/*.test.js`), `go test` for the CLI. CI workflows: `astro-testing.yml`, `javascript-testing.yml`, `go-testing.yml`, `links.yml` (lychee, config in `.lychee.toml`).
- New display strings for movies/genres live in `src/scripts/movie-labels.js` and `src/data/awesome-software-engineering-*-*.json`, not in upstream JSON.

## Change validation

Before you commit a change, always run:

- `make build` to ensure the webpage still builds
- `make test-javascript` to run the javascript unittests
- `go fmt ./...` inside `website-admin/`
- `make test` inside `website-admin/`
- `make vet` inside `website-admin/`
- `make staticcheck` inside `website-admin/`
- `make lint` inside `website-admin/`
